# Deployments using the one repo

You still use **1 GitHub repo**. These are the **deployments** (services) that use it:

| # | Platform   | What              | Repo connection     | Root / build              |
|---|------------|-------------------|----------------------|---------------------------|
| 1 | **Vercel** | Frontend (SPA)     | Same repo, 1 project | Repo root → output `dist/public` |
| 2 | **Render** | Backend (Node API) | Same repo, 1st service | Repo root, `npm run build` + `npm start` |
| 3 | **Render** | AI (Python)        | Same repo, 2nd service | Root: `ai-service`, `pip install -r requirements.txt` + `python app.py` |

So: **1 repo** → **3 deployments** (1 Vercel + 2 Render services).

Order to set up:
1. Create the **one** GitHub repo and push this codebase.
2. Connect it to **Render** twice (Backend service + AI service).
3. Connect it to **Vercel** once (Frontend).
4. Set env vars: `VITE_API_URL` and `CORS_ORIGIN` as in `.env.example` and `guide.md`.
